# Hotdog Long-Step Ablation — 阶段性报告（aggr 训练 ~70% 时刻）

## 实验设计

针对用户提出的关键假设：

> "可能是 25 → 19 → 20 → **24 → 26**... 只是 step 不够看不到第二次反弹"

我们设计了两组对照（hotdog 单场景），把训练从 7000 步拉到 **15000 步**：

| tag | config | floater 安全阀 | co-prune thr | 总 step | floater-prune 触发次数 |
|---|---|---|---|---|---|
| **aggr** | `blender_hotdog_n8_w3_aggrprune_long.yaml` | 0.25（激进）| 0.05 | 15000 | 5 次（4000 / 6500 / 9000 / 11500 / 14000）|
| **dual** | `blender_hotdog_n8_w3_dual_gs_long.yaml` | 0.08（温和）| 0.10 | 15000 | 5 次（同上） |

并在 trainer 中加了 **post-floater-prune eval hook**（+1 / +200 / +500 步），精确捕获每次 prune 后的恢复曲线。

## 关键观察

1. **每次 floater-prune 的恢复曲线高度一致**：
   * 0~1 步：基本不动（19~20 dB）
   * 200 步内：基本回到 prune 前峰值
   * 500 步内：**反弹突破 prune 前峰值 0.4~0.6 dB**

2. **峰值持续抬升**（24.85 → 25.79 → 25.89 → 26.07），LPIPS 同步下降（0.127 → 0.099）

3. **train_psnr 在每次 prune 后从 ~38 跌到 ~31，500 步内回到 ~38**，与 val_psnr 同步——说明 prune 砍掉的不全是 floater，但 densify 能在 500 步内重新填补真正需要的几何

4. **超过 baseline 7K final 23.50 dB 共 +2.57 dB**，超过早期峰值 24.85 dB 共 +1.22 dB

## 结论（待 aggr 跑完到 15000 + dual 对照）

**用户的"反弹被早停截断"假设已被强烈证实**。
* 7000 步 final = 23.50 dB 不是上限，只是处于 "Recovery +500" 阶段（step=4500 = 25.39 dB 之后的爬坡未结束）
* 15000 步内有 5 次 prune 触发，每次都贡献了 ~0.3-0.5 dB 的稳定提升

待补充：
* aggr_long 最终 test PSNR（step=15000）— 预期 **>26.5 dB**
* dual_long 最终 test PSNR（温和 prune 对照）— 用于排除 "单纯多 step 就涨" 的混淆
* 若 aggr - dual > 1.0 dB → "激进 prune + 长 step" 是真有效的提升路线
